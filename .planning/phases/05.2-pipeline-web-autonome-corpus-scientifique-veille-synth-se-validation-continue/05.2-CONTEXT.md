# Phase 05.2: Pipeline web autonome corpus scientifique veille synth se validation continue - Context

**Gathered:** 2026-03-05
**Status:** Ready for planning

<domain>
## Phase Boundary

Construire un pipeline web autonome de veille, synthese et validation continue du corpus scientifique utilise par le coach adaptatif. Le scope couvre la collecte des sources, la validation qualite/coherence, la production d'artefacts consommables par l'app, et la publication versionnee sans etat partiel.

</domain>

<decisions>
## Implementation Decisions

### Source perimeter
- Sources autorisees: guidelines + revues + consensus expertise.
- Whitelist de domaines stricte en V1 (pas d'ingestion hors domaines approuves).
- Priorite aux contenus EN; la sortie finale du coach doit rester en FR.
- Fenetre de fraicheur par defaut: 5 ans.

### Refresh orchestration
- Cadence de routine: hebdomadaire.
- Declenchement: cron only (pas de run manuel en mode nominal V1).
- Ajout de nouvelle source: backfill borne a la fenetre active.
- Echec collecte source: retry borne puis skip source, sans bloquer tout le run.

### Validation and publishing guardrails
- Gate de publication strict: pas de publication si score qualite sous seuil ou incoherence critique.
- Score qualite composite: type de source + recency + detection de contradictions.
- En cas de contradiction: arbitrage conservateur + flag de conflit.
- Pas de revue humaine obligatoire dans le flux V1.

### Synthesis and runtime consumption
- Sortie principale: JSON structure versionne.
- Granularite: synthese par principe avec provenance explicite des sources.
- Versionning: snapshots dates + rollback vers version precedente.
- Activation app: swap atomique d'un snapshot valide.
- Corpus comme guide souple: reference prioritaire pour le LLM, avec flexibilite controlee.

### Claude's Discretion
- Parametrage exact des seuils (score qualite, recency par source, retry count).
- Niveau de detail du resume humain annexe (si produit avec le JSON).
- Structure interne precise des rapports d'execution hebdomadaires.

</decisions>
